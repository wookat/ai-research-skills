# C20
_parent: **Encode Structure by Construction** (`structural_prior_encoding`)_

## Tactical pattern

This cluster treats the forward corruption/degradation process itself as the structure to encode: authors prescribe an explicit law that maps the clean target to the observed signal — a noising or mean-reverting stochastic process, a masking schedule, a physics-based formation model, or an orthogonal transform that isolates the artifact in one component — and then construct the recovery or generation operator as the principled reverse of that prescribed law. A single learnable estimator (a score/log-density-gradient network, a denoiser, a reconstruction target, or a component-wise corrector) is trained to fill in only what the forward law leaves underdetermined, so the inverse is obtained by construction rather than as a black-box corrupt→clean regression. The recurring objects are forward processes, reverse-time dynamics, decomposition components, and corruption priors; the recurring verbs are prescribe, reverse, decompose, anchor-at-observation, and reconstruct-from-multiple-views. The highest-yield variants reformulate the forward law so that the reverse unlocks otherwise-inaccessible capabilities — continuous-time integration schemes, closed-form marginals, exact likelihood, or low-variance observation-anchored initialization — instead of merely re-skinning a generic restorer.

## Step-by-Step

1. Write down the forward corruption/degradation/generation process as an explicit prescribed law — a noising kernel, a stochastic evolution, a physical formation model, or a transform that separates the quantity of interest from the artifact. The law must name a non-trivial structure of how clean becomes observed; if you can only train a generic encoder-decoder on corrupt→clean pairs, you have encoded no prior and the move silently degenerates into ordinary supervised regression that ablations expose as unremarkable.
2. Empirically verify the corruption actually lives where you claim before building on it. Isolate the structural component or quantity carrying the degradation (swap or ablate that component across clean and corrupted instances, measure its sensitivity), so the prior rests on a demonstrated equivalence rather than an assertion; an unvalidated 'this component carries the corruption' claim is confoundable and will not survive review.
3. Construct the recovery or generation operator as the principled reverse of the prescribed forward law, so that a single learnable estimator fully specifies the inverse (a reverse-time process from the forward dynamics, an energy-gradient direction, or a component-wise correction). Prefer a reformulation whose reverse unlocks capabilities that were structurally impossible under the original framing — a continuous toolkit, closed-form marginals, exact density, or new sampling schemes — rather than one that only relabels an existing pipeline.
4. Anchor the process to the actual observation wherever possible: set the endpoint, equilibrium, or initialization to the corrupted observation, or aggregate complementary information across multiple corrupted views, instead of inheriting an uninformative reference state wholesale. This keeps the reverse process low-variance and faithful to structure; declining to let the corruption itself anchor the process forfeits the main advantage of having modeled it.
5. Demonstrate the encoded structure — not the backbone — is load-bearing. Ablate the prior against an equal-capacity baseline, test every claimed generality (plug-in, universal, attack-agnostic) across more than one base model, dataset, or corruption type, and report fidelity gains attributable specifically to the reformulation; if improvements cannot be separated from a powerful pretrained backbone or the breadth claim is tested on a single instance, reviewers will credit capacity, not the prior.

## Differentiation within parent

Unlike sibling cluster 16, which encodes invariance to a transformation group by averaging a base procedure over a small representative frame so a symmetry property holds by construction, this cluster encodes the forward corruption trajectory and builds the reverse of it — the prior here is a degradation/generation dynamics, not a static group action to be respected. It also differs from sibling cluster 13, which encodes relational topology by repurposing learned pairwise interaction weights as link-likelihood scores: cluster 13's structure is a graph over co-existing entities, whereas this cluster's structure is a temporal/generative map between a clean state and its corrupted observation. The distinguishing object is therefore a prescribed forward process — noising, masking, physical degradation, or an artifact-isolating transform — whose inverse becomes the model, rather than a symmetry to enforce or a relational graph to infer.

## When to pick this one

Pick this cluster when the structure you can actually write down is a forward corruption/degradation/generation process — you can state, physically model, or decompose how a clean signal becomes the observed one — so recovery or generation can be constructed as its reverse. If instead the known structure is an invariance to a transformation group or a relational graph among co-existing entities, a sibling cluster is the better instantiation.

## Tactical failure mode

Papers attempting this tactic fail when the corruption-structure prior is asserted but never isolated as the actual source of gains. The recurring shape takes three forms: the "structure" is really a generic architecture or a powerful pretrained backbone doing the work, so same-capacity ablations cannot attribute any improvement to the prior; or the structural reformulation is theoretically clean — an equivalence proof, a physical decomposition — but the empirical delta over the base method is marginal while the advertised generality (plug-in, universal, attack-agnostic) is demonstrated on a single backbone or dataset; or the physical/derivational grounding is presented without rigor and with missing baselines, missing citations, or undersized evaluation. The common defect is that the encoded forward process is never shown to be load-bearing relative to capacity. A candidate author should self-check: can you ablate the prior against an equal-capacity model, isolate its contribution from the backbone, and confirm the claimed breadth on more than one model, dataset, or corruption type?

## Examples
### Oral lessons
- Lifting a discrete corruption ladder to a continuous-time forward law unlocks a whole mathematical toolkit — numerical integration schemes and exact likelihood — that was structurally inaccessible in the discrete framing, making the continuous limit operationally enabling rather than merely elegant.
- A long-understood classical decomposition can cleanly partition the quantity of interest from the corrupting artifact into independently correctable components, so the separation is achieved with no new theoretical machinery at all.
- Empirically prove the corruption lives in one structural component — for instance by swapping that component between clean and corrupted instances — before promoting it from a computational convenience into a built-in, reusable prior.
- When clean references are unavailable, reframe a set of corrupted observations as a distribution to match rather than elements to align, so a correspondence-free loss recovers clean structure without any registration or paired targets.
- When corruption severs non-local structure that surviving local context cannot recover, reconstruct the target by aggregating complementary information across multiple corrupted views instead of relying on a single corrupted instance.
- A corruption-and-reconstruction generative factorization can replace a default ordered factorization and remove its structural limitations while matching performance at scale, revealing that emergent properties come from the objective and model family, not the processing order.

### Reject lessons
- Claiming an architecture carries a domain-matched structural prior while shipping a generic off-the-shelf encoder-decoder collapses the move into ordinary supervised denoising that ablations and weak-baseline comparisons quickly expose.
- An elegant equivalence proof (corruption-recovery equals descent on an energy landscape) earns nothing when the empirical gain over the base method is marginal and the advertised plug-in generality is shown on only a single backbone.
- Deriving partition labels from a first-principles physical model is undercut when the derivations lack rigor, baselines and ablations are missing, and citation hygiene fails.
- If the structural signal you minimize rides on a powerful pretrained backbone, failing to isolate the prior's contribution from sheer backbone capacity leaves reviewers unable to credit the idea and unconvinced by the motivation for the chosen backbone.
