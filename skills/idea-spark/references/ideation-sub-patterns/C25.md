# C25
_parent: **Decompose and Delegate to Solvers** (`decompose_and_delegate`)_

## Tactical pattern

Have the learned model emit a structured, machine-checkable intermediate artifact at the decomposition boundary — an executable program, a formal domain/world model, a relational-or-constraint graph, a concrete state waypoint, an interleaved reason-act/tool-call trace, or a typed inter-agent deliverable — in place of a final answer, then route that artifact to a sound solver (interpreter, classical planner, constraint solver, MCTS/RL optimizer) or back through the model's own verify-and-revise loop. The load-bearing move is closing the loop with the solver's verifiable output — execution results, validator verdicts, behavioral consensus across independently sampled candidates, or terminal-answer confidence — and using that signal for credit assignment, candidate selection, or in-place correction applied DURING construction rather than only post-hoc. The learned component is confined to extraction, enrichment, planning, and interfacing; structured or symbolic resolution is offloaded to the component that handles it exactly. The recurring innovation is converting the intermediate representation from advisory output into a load-bearing object that something downstream can mechanically check.

## Step-by-Step

1. Identify which sub-problem the learned model handles unreliably and which a sound solver handles exactly — combinatorial search, precise computation, structural path-finding over a formal space, or pinpoint localization within a large corpus. Frame this as a responsibility boundary between comparative advantages, not a capability gap to be trained away. If you cannot name a structurally distinct bottleneck and instead merely chain several generic agents, the decomposition will read as incremental assembly of off-the-shelf parts.
2. Design the intermediate artifact that crosses this boundary to be inspectable and machine-checkable — typed and schema-conforming, executable, or formally verifiable — so the handoff carries an object a solver can consume, not free-form prose the next stage merely reads. An advisory plan with no verification closure leaves the artifact non-load-bearing and collapses the contribution to ordinary sampling-plus-filtering. The representation should align with the form the task's own specification or examples already use, so each piece is independently checkable.
3. Route the artifact to the matched solver (or to the model's own critique loop) and close the loop with the solver's verifiable output: execution results, validator verdicts, behavioral consensus among independently sampled candidates, or terminal-answer confidence. Apply this feedback mid-construction so it can assign credit to individual sub-decisions, select among candidates, or trigger in-place correction — not only filter complete outputs after the fact. Where self-generated evaluation signals are known to be unreliable, substitute an external or consensus-based oracle rather than trusting the model to grade itself.
4. Ablate the verification-feedback step to prove it is load-bearing: demonstrate that removing it returns performance to the undecomposed baseline. Guard against the confound where apparent gains actually come from extra training data, more epochs, or the base model's pre-existing knowledge — isolate the structure's marginal contribution explicitly, or reviewers will discount the gains as unattributable.
5. Account for the cost the delegation introduces — solver latency, multiplied model calls, or the quality of any pipeline that supplies the artifact's supervision — and validate beyond toy instances. An unexamined NP-hard solver cost, an unvalidated data-generation or retrieval component, or an elicitation scheme tuned to a single benchmark will sink an otherwise elegant decomposition. State the disclosure honestly when the loop depends on a privileged signal (e.g., available test cases) that real deployments lack.

## When to pick this one

Instantiate the methodology as this tactic when the unreliable sub-problem has a sound external solver or a mechanically checkable oracle available — an interpreter, classical planner, constraint solver, search/RL routine, or executable test suite — and the learned model can emit a structured artifact that the solver consumes and returns verifiable feedback on. The signal is "I can name a checker that grades the intermediate object," not just "I can split the task into stages."

## Tactical failure mode

The recurring failure is assembling a decompose-and-delegate pipeline whose components are all off-the-shelf — role-agents, prompts, existing engines, retrieval modules — so that no single piece is shown to be both novel and load-bearing. The intermediate artifact stays advisory (read by the next stage but never closed by a verification signal), reducing the system to sampling-plus-filtering or to a re-skin of an already-published neighbor that performed the same routing move. When the delegated solver is sound but expensive, authors omit the latency and scalability accounting that the delegation demands, or they lean on a benchmark-tailored prompt/elicitation scheme that reads as an ad-hoc hack rather than a transferable mechanism. Most damaging, they skip the ablation that isolates the decomposition's marginal contribution from extra data, prior model knowledge, or larger backbones, leaving reviewers unable to attribute the gains to the proposed structure. Self-check: can you name the one component whose removal collapses performance to the baseline, and have you empirically shown that collapse while ruling out confounds?

## Examples
### Oral lessons
- Using the same generative mechanism both to produce candidate intermediate states and to score them eliminates the need for a separately trained evaluation oracle — viable precisely when self-scoring is reliable enough to steer a search.
- Interleaving deliberation steps and external-query steps within a single output trajectory (rather than running them as separate streams) creates the feedback loop that grounds each reasoning step and suppresses unsupported conclusions.
- Expressing sub-goals as concrete, independently verifiable state configurations rather than symbolic descriptions lets learned sub-tasks recombine for compositional structures never seen in training.
- Treating a solver's intermediate by-products — synthesized lemmas, discovered composite operations, iterative repair diffs — as persistent reusable capital turns each individual solve into cumulative library growth at near-zero marginal cost.
- Enforcing typed, schema-conforming artifacts at every inter-component handoff breaks the cascading-error chain that free-form natural-language passing silently propagates.
- When a system's own self-generated evaluations are empirically unreliable, replacing them with behavioral consensus among independently sampled candidates is a more robust correctness proxy than any self-critique.

### Reject lessons
- A decomposition that merely chains off-the-shelf agents or engines, with no single component demonstrated to be both novel and load-bearing, reads as engineering assembly rather than a contribution.
- If the 'key' prompt or elicitation scheme is tuned to one benchmark and not validated across settings, reviewers treat it as a dataset-specific hack instead of a transferable mechanism.
- Delegating a sub-problem to a sound but costly solver (NP-hard search, materialized intermediate state) sinks the contribution when latency and scalability are left unexamined.
- Without an ablation isolating the decomposition's marginal effect from extra data, more epochs, or the base model's prior knowledge, the reported gains read as confounded and unattributable.
- Re-deriving a routing move that a closely neighboring published system already demonstrated, while showing only marginal gains, fails the novelty bar regardless of engineering polish.
- A self-improvement or decomposition loop whose reliability hinges on an unvalidated data-generation, retrieval, or filtering component invites rejection when error propagation across iterations is never analyzed.
