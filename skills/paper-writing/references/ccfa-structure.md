
# Paper Writing（论文写作）

先定故事，再写句子。输入：`evidence_summary.md`、`hypothesis_tree.md`、idea 评审卡、目标会议。

## Step 0 — 故事定型（写任何正文前）

用五句话写出论文故事，人审后冻结：

1. 问题：<领域> 中 <具体问题> 重要因为 <理由>
2. 缺口：现有方法因为 <假设/设计> 在 <条件> 下失败
3. 洞察：我们发现 <关键洞察>
4. 方法：据此提出 <机制>
5. 证据：在 <数据集> 上 <核心结果>，消融证明 <组件必要性>

## 结构蓝图（按目标会议模板落 LaTeX）

- **Abstract**：缺口→洞察→方法→最强数字，150 词内，每个主张都要有实验对应
- **Introduction**：4-6 段，每段一个信息点：问题重要性 / 现有方法与失败点 /
  关键洞察（本文最重要的一段）/ 方法概述 / 贡献列表（3 条，每条可被单独验证）。
  Figure 1 = teaser 图，直观展示"失败现象 + 本方法修复"
- **Related Work**：按"与本文的关系"组织而非罗列；scoop-check 圈出的最接近工作必须正面对比
- **Method**：动机→设计→为什么这样设计，逐小节；符号表先行；配 pipeline 图
- **Experiments**：主表 → 消融 → 分析（含失败案例与被证伪假设的洞察）→ 效率。
  每张表格在正文有一句"该表证明了主张 X"
- **Conclusion + Limitations**：诚实的 limitation 是加分项，写审稿人反正会发现的那些

## 段落级写作法（逐段执行）

1. 一段只说一件事；首句即段旨
2. 句间关系显式（因果/转折/递进/举例），杜绝并列堆砌
3. 新术语先定义后使用；全文术语唯一（同物不换名）
4. 每写完一节做**反向大纲**：抄下各段首句，检查能否映射回论文主线；映射不上的段落删或改
5. 禁止"incremental patching"式写法（"我们在 X 上加了 Y 又加了 Z"）；始终从洞察出发叙述

## 主张-证据硬约束

维护 claim-evidence map：`Claim | Evidence(实验ID/表号) | Status`。
Abstract 和 Introduction 中每个主张必须 Status=supported；撑不住的主张降级措辞或删除，
**改主张不改数据**。

## 图表规范

- 图：矢量输出、全文统一字号/配色、图注自含（不看正文能懂）
- 表：最优加粗、次优下划线、报均值±方差、行列顺序有逻辑
- Figure 1 与 pipeline 图值得花全文 20% 的打磨时间——审稿人前 5 分钟只看图表和 abstract

## 收尾自审（提交给 reviewer-simulation 前）

五维自查并逐项修复：贡献清晰度 / 写作流畅度 / 实验强度 / 评测完备性 / 方法合理性。
另查：引用格式统一、所有交叉引用有效、页数限制、匿名化要求、checklist（NeurIPS 类会议）。

## 纪律

- 一次只重写一节；每节完成即做反向大纲，不积压。
- 禁止编造引用；每条 bib 与真实论文核对过标题作者。
- 中文思考、英文成文时，警惕直译腔：长定语拆短句、被动改主动。
